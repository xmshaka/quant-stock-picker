## 已完成

### 1. 回测一致性 P0
- 默认 K线买卖点改为 `signals_executed`。
- `signals_raw` 只作为可选叠加层，不参与默认统计。
- 回测统计、交易明细、K线点位统一到实际成交事件源。
- 新增一致性校验：`buy_count/sell_count/trade_details/signals_executed` 必须匹配。

### 2. 数据口径修复
- `_fetch_ohlcv()` 默认改为不复权 `adjust=""`。
- 明确原则：撮合/默认K线使用不复权，前复权仅用于趋势展示。
- 解释并修复 603993 2026-05-13 价格异常：旧值 20.75 来自前复权 open 20.704 × 1.002 滑点；修复后使用不复权 open 20.99，成交约 21.03。

### 3. 成本与风控
- 成本模型齐全：佣金、最低佣金、印花税、过户费、滑点。
- 风控：总仓 90%，单票 20%，A股 100 股整数手。
- Backtrader 和单股模拟路径均记录实际成交点和明细。

### 4. trades schema 标准化
- 新增 `STANDARD_TRADE_COLUMNS`。
- 新增 `trade_details_to_frame()`，`trades.parquet` 现在以 `trade_details` 为准，不再用 TradePoint 反推。
- 新增 `validate_trade_schema()`。
- 标准字段包含：run_id/symbol/date/signal_date/exec_date/action/event_type/source/reason/rule_name/signal_price/price/exec_price/shares/amount/commission/stamp_duty/transfer_fee/slippage/slippage_rate/commission_total/avg_cost/cost/pnl/pnl_pct/holding_days/cash_after/position_after/position_shares/stop_loss/take_profit。
- 旧版历史记录读取时自动 legacy normalize，缺失成本字段补 0，避免审计异常。

### 5. K线图买卖点样式
- 买卖点改为固定上下轨道，不遮挡任何K线柱/影线。
- B/S 标志外置。
- 加虚线连接到对应K线 low/high。
- 价格与 B/S 标志分轨显示。
- 当前用户确认：K线好了。

### 6. 策略回测页面
- 指标卡「交易」只显示 `N轮`，避免卡片高度不一致。
- 自定义股票池、策略、top_n、lookback、资金等参数加 durable session state，切换页面后不再重置到全A/低波价值。
- 回测完成后自动保存 run，无需手动点持久化。
- 持久化 config 保存 `pool_mode` 和 `symbols`。

### 7. 历史回测记录页面
- 新增 `dashboard/pages/9_回测记录.py`。
- 新增 `list_backtest_runs()` / `load_backtest_run()`。
- 页面支持：历史run列表、指标、schema审计、一致性检查、交易流水、权益曲线、K线复盘、config/report/signals查看。
- 历史页取消缓存，进入即读取最新目录。
- 增加删除功能：软删除到 `data/backtest_runs/.trash/`。
