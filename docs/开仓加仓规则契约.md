# 开仓 / 加仓规则契约（专家审计版）

更新时间：2026-06-21 13:55 GMT+8

## 1. 基本立场

本项目定位是 A 股短线择时选股系统，目标不是拟合单次回测胜率，也不是为已有结果找解释，而是提高实盘买卖机会点准确度。

因此规则开发顺序必须是：

```text
交易假设 → 买点类型 → 主触发 → 确认项 → 否决项 → 置信度 → 执行/仓位 → 回测验证
```

禁止流程：

```text
看到某个 run 多/少了买点 → 临时加冷却/阈值/条件 → 让结果看起来合理
```

当前开仓执行契约：

```text
watch / observe_only / weight=0
  只保留 raw signal，不进入 executed/trades，不实际开仓

candidate / reduced_or_pending / weight=0.5
  允许开仓但降仓执行

standard / strong / weight=1.0
  标准执行
```

所有开仓仍必须满足 T+1、成本、100股整数手、大盘择时调制和单票20%硬上限。

## 3. 置信度契约

`TradePoint.confidence` 必须从“展示字段”升级为“执行契约字段”。

### 3.1 开仓置信度原则

专业原则：

```text
低置信度信号不能和高置信度信号同等开仓。
```

当前第一阶段执行分层（阈值待历史分布验证后固化）：

```text
confidence < 0.50
  watch / observe_only
  只观察，不交易；信号保留在 signals_raw，不进入 signals_executed/trades

0.50 <= confidence < 0.65
  candidate / reduced_or_pending
  允许降仓开仓；仓位权重 confidence_weight = 0.5

0.65 <= confidence < 0.80
  standard / standard_entry
  标准仓位权重 confidence_weight = 1.0

confidence >= 0.80
  strong / strong_entry
  高置信度开仓；当前仓位权重仍为 1.0，不突破单票20%上限
```

执行金额：

```text
entry_alloc = cash × position_pct_per_entry × market_multiplier × confidence_weight
entry_alloc = min(entry_alloc, total_value × max_single_pct_hard_20%)
shares = floor(entry_alloc / exec_price / 100) × 100
```

注意：candidate 降仓不是“放松交易”，仍必须满足大盘择时、T+1开盘撮合、成本、滑点和可交易性约束。

### 3.2 加仓置信度原则

加仓必须比开仓更严格。

建议契约：

```text
allow_add = (
    current_position_cost_adjusted_pnl > 0
    and original_entry_structure_not_broken
    and same_entry_model_confirmed_again
    and new_signal.confidence >= add_threshold
    and new_signal.confidence >= initial_signal.confidence + confidence_delta
    and add_count < max_add_times
    and post_add_single_position_pct <= 20%
)
```

禁止：

```text
亏损摊平式加仓
不同买点模型混合加仓
低置信度重复 BUY 自动加仓
跌破原始买点结构后加仓
```

## 4. 买点类型契约

买点不能由“任意指标凑够 N 个”直接触发。每个买点必须归属于一个模型。

### 4.1 趋势延续买点 trend_continuation

适用策略：`trend_momentum`

交易假设：强势趋势仍在延续，短线资金继续推高。

主触发：

```text
接近20日新高
MA20 > MA40
5日/20日动量为正
```

确认项：

```text
量比适度放大
成交额足够
收盘接近日高
大盘非防御
```

否决项：

```text
RSI极端过热
高位爆量滞涨
长上影冲高回落
涨停封死不可成交
```

加仓：只允许盈利后顺势确认加仓，最多一次。

### 4.2 趋势回调低吸 pullback

适用策略：`pullback`

交易假设：中短趋势未破，回调到支撑/均线/布林下沿附近后恢复。

主触发：

```text
MA20 > MA40
距20日高点回撤 5%~15%
价格接近 MA20 / 前平台 / 布林下轨
```

确认项：

```text
RSI < 35 或 RSI低位拐头
缩量回调优先
收盘不能接近日低
支撑未破
```

否决项：

```text
跌破20日低点
放量长阴
MA20走平/下弯
近10日无有效上升趋势
大盘防御
```

加仓：默认不加仓；除非首次仓位已盈利且回踩不破原始结构。

### 4.3 平台突破 breakout

适用策略：`breakout`

交易假设：横盘蓄势后资金放量突破平台。

主触发：

```text
10~20日振幅收窄
突破平台上沿
成交量显著放大
```

确认项：

```text
收盘站上平台上沿
换手率适中放大
MACD改善
大盘非防御
```

否决项：

```text
突破当天长上影
放量冲高回落
次日低开跌回平台
成交额不足
```

加仓：只允许突破后回踩平台不破再加一次。

### 4.4 超跌反弹 oversold_rebound

该模型风险最高，默认不建议作为 `balanced` 宽口径触发。

如保留，主触发应极严格：

```text
RSI < 25，最好 < 20
布林下轨外
短期跌幅过大
```

确认项：

```text
长下影或次日不创新低
缩量止跌或放量承接
成交额足够
```

否决项：

```text
空头排列
连续跌停
放量破位
基本面/公告风险
大盘防御
```

加仓：禁止。

## 5. balanced 定位契约

`balanced` 长期不应作为“任一形态 + 任意2/6指标共振即可交易”的独立策略。

推荐定位：

```text
组合器 / 路由器
```

即：

```text
分别运行 trend_momentum / pullback / breakout / oversold_rebound（如启用）
每个信号带 entry_model、confidence、main_trigger、confirmations、veto_checks
balanced 只负责融合排序、仓位建议和风险标签
```

过渡期要求：

```text
balanced 的每个 BUY 必须标注 entry_model。
如果无法归类，不应交易，只能展示为未分类候选。
```

## 6. 20260620_172722_balanced 审计结论

该 run 中 600143 三个 BUY 的置信度：

```text
2026-03-25 BUY confidence=0.4903
2026-03-31 BUY confidence=0.4205
2026-06-03 BUY confidence=0.6603
```

其中前两个新增 BUY 不是加仓，`event_type=BUY`，没有 `ADD`。

暴露的问题不是“某个日期要不要临时过滤”，而是：

```text
执行层没有使用 confidence，导致低置信度候选与较高置信度信号被同等执行。
```

后续代码修改必须围绕“置信度进入执行契约”展开，而不是围绕单个日期拟合。
