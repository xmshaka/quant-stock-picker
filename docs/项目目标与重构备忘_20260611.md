## 一、项目目标重新定义

本项目目标是开发一个适合 **A股 20 个交易日内行情** 的短周期择时选股分析系统。

核心定位：

```text
A股20日短周期择时选股分析系统
盘后筛选 + 盘后总结 + 观察池/持仓提醒 + 后续盘中盯盘
辅助人工决策，不接自动交易
```

当前不做自动交易，所有信号只用于分析、提醒和人工决策。

## 二、当前版本主要问题

### 1. 缺少大盘择时层

当前系统主要对大A所有符合过滤条件的股票做个股因子打分，保证相对稳定，但缺少市场环境判断。

缺失的大盘择时因子包括：

```text
市场波动率
涨跌家数比
两市成交额
站上均线股票占比
涨停/跌停家数
炸板率
连板高度
指数趋势
市场宽度
市场流动性
```

影响：

```text
无法判断当前该不该开仓
无法判断仓位高低
无法判断适合趋势、反转、低波还是防御方案
无法根据市场状态切换因子权重和策略方案
```

---

### 2. 当前因子体系不适合 20 日内短周期择时

当前部分因子更偏中长期价值投资，对 20 个交易日内涨跌解释力不足。

主要问题：

```text
PE_TTM 对亏损股、周期股、高成长股失真严重
动量因子重复度高
部分趋势/技术因子表达高度重叠
缺少总市值、流通市值、成交额、流动性等A股主流短周期因子
缺少相对指数强弱、量价结构、波动风险、市场风格因子
```

需要重新梳理适合本项目目标的因子方案和策略方案。

---

### 3. 回测缺少记录管理

当前回测更像一次性计算结果，缺少结构化记录。

需要补充：

```text
run_id
参数快照
因子版本
策略版本
数据版本
原始信号记录
实际成交记录
权益曲线
持仓变化
回测报告
历史回测对比
```

尤其要解决：

```text
回测记录与K线图买卖点多次不一致
```

默认原则应调整为：

```text
K线图默认展示实际成交记录
原始规则信号只作为可选叠加层
绩效统计、交易明细、K线买卖点必须来自同一事件源
```

---

### 4. 缺少每日盘后自动总结与提醒

每日数据更新后，系统应自动完成：

```text
今日行情总结
市场状态判断
建议仓位
适用策略方案
全市场候选股票
观察池信号提醒
持仓池风险提醒
飞书/微信自动发送，可配置
```

---

### 5. 后续需要盘中盯盘信号

盘中盯盘作为第二阶段能力。

优先对象：

```text
持仓池
观察池
昨日候选池
重点跟踪股票
```

提醒范围：

```text
突破买点
跌破止损
接近止盈
放量异动
跌破关键均线
接近涨停/跌停
炸板/回封
```

明确：只提醒，不自动交易。

---

### 2. 市场状态分类

先定义 5 类：

| 市场状态 | 说明 | 仓位建议 | 适合策略 |
|---|---|---:|---|
| 强势趋势 | 指数MA20上方，宽度强，成交放大 | 70%-90% | 趋势动量、强势回踩 |
| 温和上行 | 指数偏强，宽度中等 | 50%-70% | 质量成长、趋势回踩 |
| 震荡 | 指数无趋势，宽度反复 | 30%-50% | 超跌反转、低吸 |
| 弱势下跌 | 指数MA20下方，宽度弱 | 0%-30% | 防御、少量反弹 |
| 极端风险 | 大跌、跌停多、成交萎缩 | 0%-10% | 空仓观察 |

输出结构示例：

```python
MarketRegime(
    date="2026-06-11",
    regime="震荡",
    risk_level="中",
    suggested_position=0.3,
    preferred_schemes=["reversal", "low_vol_pullback"],
    forbidden_schemes=["breakout_chase"],
    reason="指数低于MA20但涨跌家数改善，成交未放大"
)
```

### 6. 质量/估值因子降权处理

保留但不作为短期主导：


PE_TTM 处理原则：

```text

PE > 200 winsorize
优先使用 EP = 1 / PE

```

### 7. 因子处理规范


## 六、策略方案重构

建议先建立 5 个明确风格方案。

### 1. 趋势动量方案

适用：

```text
强势趋势 / 温和上行
```

因子：

```text
relative_strength_20d
ma20_slope
ret_20d
amount_ratio_5d
new_high_60d_distance
volatility_20d 负向
float_mv 稳定偏好
```

买点：

```text
回踩 MA10/MA20 不破
放量重新站上短均线
MACD翻红
```

卖点：

```text
跌破 MA20
ATR止损
放量长阴
```

---

### 2. 超跌反转方案

适用：

```text
震荡 / 弱势修复
```

因子：

```text
rsi14 低
boll_position 低
max_dd_20d 高
ret_5d 负向
amount_ratio_5d 放大
基本流动性达标
```

买点：

```text
RSI进入超卖后回升
布林下轨附近企稳
缩量下跌后放量阳线
```

卖点：

```text
反弹到 MA20
RSI过热
ATR跟踪止盈
```

---

### 3. 低波质量方案

适用：

```text
弱势市场 / 防御
```

因子：

```text
roe
gross_margin
低波动
低最大回撤
中大市值
成交稳定
pb/ep 辅助
```

买点：

```text
市场风险不高
价格重新站上 MA20
成交温和放大
```

---

### 4. 小中盘活跃方案

适用：

```text
市场情绪强、成交活跃
```

因子：

```text
float_mv 适中
amount_20d_avg 足够
turnover_ratio
volume_ratio
relative_strength_10d
板块强度
```

过滤：

```text
排除极低流动性
排除连续暴涨高风险
排除跌停/炸板高风险
```

---

### 5. 观察池/持仓专用信号方案

```text

有没有破位
```


### 6. K线图一致性原则

默认：

```text
K线图 = signals_executed
回测记录 = trades.parquet
绩效统计 = trades.parquet + equity.parquet
```

可选叠加：

```text
规则原始信号 = signals_raw
```

验收标准：

```text
result.buy_count == K线图 BUY 点总数
result.sell_count == K线图 SELL 点总数
result.trade_details 行数 == K线买卖点明细行数
每个 TradePoint 都能在 trade_details 中找到同日期/同股票/同方向/同股数记录
原始规则信号不参与默认 K线买卖次数统计
如显示原始规则信号，必须用不同图形/颜色/图例标记为“规则信号”
```
