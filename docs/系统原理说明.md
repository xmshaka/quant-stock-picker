## 二、因子体系（5大类15因子）

系统目前使用 **15 个核心因子**，覆盖五大维度：

| 类别 | 因子名 | 中文名 | 因子方向 | 含义 |
|------|--------|--------|---------|------|
| **技术因子** | `rsi14` | RSI14 | -1 (越低越好) | 相对强弱指标，衡量超买超卖 |
| | `boll_position` | 布林带位置 | +1 (越高越好) | 价格在布林带中的相对位置 |
| | `volatility_20d` | 20日波动率 | -1 (越低越好) | 20日收益率标准差 |
| | `max_dd_60d` | 60日最大回撤 | -1 (越低越好) | 60日内最大跌幅 |
| **情绪因子** | `north_hold_change` | 北向资金20日变化 | +1 | 北向资金持仓变化趋势 |
| | `turnover_ratio` | 换手率比率 | +1 | 成交量/流通股本 |
| | `volume_ratio` | 量比 | +1 | 当前成交量/近期平均 |
| **估值因子** | `pe_ttm` | 市盈率TTM | -1 (越低越好) | 滚动市盈率 |
| | `pb` | 市净率 | -1 (越低越好) | 市净率 |
| | `ep` | 盈利收益率 | +1 | EPS/股价 |
| **动量因子** | `momentum_5d` | 5日动量 | -1 (偏反转) | 近5日涨跌幅 |
| | `momentum_20d` | 20日动量 | -1 (偏反转) | 近20日涨跌幅 |
| | `momentum_60d` | 60日动量 | 0 (中性) | 近60日涨跌幅 |
| | `reversal` | 反转因子 | +1 | 短期反转信号 |

> **因子方向**：`+1` = 越高越好，`-1` = 越低越好，`0` = 中性观察

## 三、核心概念："得分"与"强度"

### 3.1 综合得分 (Score)

**综合得分**是系统对一只股票在当前截面的多因子综合评估结果，范围约为 **-3 ~ +3**（理论上无界，但实际分布在此范围内）。

**计算步骤：**

```
步骤1: 对每个因子计算截面 Z-score
       Z = (个股因子值 - 全市场均值) / 全市场标准差

步骤2: 根据行情分类，调整因子权重和方向
       - 震荡市: 反转因子权重↑，动量因子反向
       - 强势上涨: 动量因子正向跟随，RSI极端衰减
       - 弱势上涨: 动量因子弱化
       - 强势下跌: 反转信号弱化
       - 弱势下跌: 反转因子权重↑

步骤3: 加权求和
       Score = Σ(Zi × Wi_effective) / Σ|Wi|
```

**得分解读：**

| 得分区间 | 含义 |
|---------|------|
| ≥ +1.0 | 🟢 明显优于市场平均，加分项 |
| +0.3 ~ +1.0 | 🟡 小幅优于平均 |
| -0.3 ~ +0.3 | ⚪ 接近市场平均 |
| -1.0 ~ -0.3 | 🟠 小幅劣于平均 |
| ≤ -1.0 | 🔴 明显劣于平均，扣分项 |

### 3.2 信号强度 (Strength)

**信号强度**是将得分映射到 **0-10** 的直观等级，用于快速判断信号的可靠程度。

**计算公式：**

```python
# 买入信号
strength = min(10, max(0, z × 2 + 5))
# 其中 z = (个股得分 - 市场得分均值) / 市场得分标准差

# 卖出信号
strength = min(10, max(0, -z × 2 + 5))
```

**强度分级：**

| 强度 | 颜色 | 含义 | 操作建议 |
|------|------|------|---------|
| 8-10 | 🟢🟢 | 极强信号 | 重点关注，可加大仓位 |
| 6-7.9 | 🟢 | 强信号 | 值得考虑 |
| 4-5.9 | 🟡 | 中等信号 | 可作为候选 |
| 2-3.9 | 🟠 | 弱信号 | 仅供参考 |
| 0-1.9 | ⚪ | 极弱信号 | 不显示（低于阈值） |

> 系统默认 `min_strength=2.0`，低于此值的信号不显示。

## 四、行情分类自适应（Market Regime）

这是本系统区别于传统多因子模型的**核心创新点**。

### 4.1 五种行情分类

系统根据 **20日动量绝对值** 和 **均线排列** 将每只股票归类：

| 行情分类 | 判定条件 | 策略倾向 |
|---------|---------|---------|
| **强势单边上涨** | 20日动量 > 阈值 且 MA5>MA10>MA20 且 价格>MA5 | 趋势跟随，动量加分 |
| **弱势单边上涨** | 20日动量 > 阈值 但 均线未多头排列 | 动量弱化，等待确认 |
| **震荡整理** | 20日动量绝对值 < 阈值×0.5 | 反转策略，低买高卖 |
| **弱势单边下跌** | 20日动量 < 阈值 但 均线未空头排列 | 反转因子强化 |
| **强势单边下跌** | 20日动量 < 阈值 且 MA5<MA10<MA20 且 价格<MA5 | 跟随下跌，RSI超卖保护 |

### 4.2 因子自适应规则

以 **RSI14** 为例，在不同行情下的处理方式完全不同：

```
震荡整理  →  反转逻辑: 低RSI加分（抄底）
强势上涨  →  跟随逻辑: 高RSI加分（追涨），但≥75时衰减50%（追高风险）
弱势上涨  →  中性: 不参与打分
强势下跌  →  跟随逻辑: 高RSI扣分（持续弱势），但≤25时衰减50%（超卖保护）
弱势下跌  →  中性: 不参与打分
```

**动量因子** 的自适应：

```
强势单边(涨/跌)  →  权重 ×1.3 (趋势跟随)
弱势单边(涨/跌)  →  权重 ×0.3 (弱化)
震荡整理         →  反向 ×(-1) (反转策略)
```

**反转因子** 的自适应：

```
震荡整理        →  权重 ×1.2 (最重要)
弱势单边       →  权重 ×0.8 (等待反转)
强势单边       →  权重 ×0.2 (几乎不用)
```

## 五、IC分析与因子有效性检验

### 5.1 什么是IC？

**IC (Information Coefficient，信息系数)** 衡量因子的预测能力：

```
IC = Corr(今日因子值, 未来N日收益率)
```

- **IC > 0**：因子与收益正相关，因子越高未来收益越高
- **IC < 0**：因子与收益负相关，因子越高未来收益越低
- **IC ≈ 0**：因子无预测能力

系统同时计算 **Pearson IC**（线性相关）和 **Rank IC**（秩相关，更稳健）。

### 5.2 IC统计指标

| 指标 | 含义 | 判断标准 |
|------|------|---------|
| **IC均值** | 因子预测能力的平均表现 | >0.03 为有效，>0.05 为优秀 |
| **IC标准差** | 预测稳定性 | 越小越稳定 |
| **IR (信息比率)** | IC均值 / IC标准差 | >0.5 为有效，>1.0 为优秀 |
| **正IC占比** | IC>0的交易日比例 | >55% 为有效 |
| **有效天数** | 有统计意义的交易日数 | 越多越好 |

### 5.3 IC衰减分析

系统检验因子在不同预测期的有效性：

```
预测期=1日  →  短期动量/反转
预测期=5日  →  周度预测（默认）
预测期=10日 →  双周预测
预测期=20日 →  月度预测
```

有效因子的 IC 通常随预测期延长而衰减。如果某因子在20日预测期仍有较高IC，说明其具有**中长期预测能力**。

## 六、因子去重方法（高相关性处理）

### 6.1 为什么要去重？

当两个因子高度相关时（|相关系数| > 0.7），它们提供的是**重复信息**，会导致：
- 模型过度拟合
- 某些维度的信息被过度加权
- 信号稳定性下降

### 6.2 去重步骤

**第一步：截面相关性分析**

取最新一个交易日的截面数据，计算所有因子两两之间的相关系数矩阵：

```python
corr_matrix = factor_df.pivot(index='symbol', columns='factor_name').corr()
```

**第二步：识别高相关因子对**

系统标记 |相关系数| > 0.7 的因子对：

```
示例高相关对：
  • momentum_5d ↔ momentum_20d: +0.85  (同为动量，只是周期不同)
  • pe_ttm ↔ ep: -0.92  (PE和EP互为倒数)
  • turnover_ratio ↔ volume_ratio: +0.78  (都是成交量相关)
```

**第三步：决策保留或合并**

| 情况 | 处理方式 | 示例 |
|------|---------|------|
| 同类型不同周期 | 保留周期更长的 | 保留20日动量，去掉5日动量 |
| 互为倒数/衍生 | 保留其中一个 | 保留EP，去掉PE |
| 信息来源不同但结果相似 | 根据IC择优 | 保留IC更高的 |
| 高负相关 | 通常保留（提供反向信息）| 动量和反转可同时保留 |

**第四步：分组收益验证**

对疑似冗余的因子，进行**分组收益分析**：

```
将股票按因子值分为5组（Q1-Q5）
计算每组未来N日的平均收益

有效因子特征：分组收益呈单调趋势
  Q1(最低) < Q2 < Q3 < Q4 < Q5(最高)  →  正向因子
  Q1(最低) > Q2 > Q3 > Q4 > Q5(最高)  →  反向因子

冗余因子特征：两组收益曲线几乎重合
```

### 6.3 当前系统因子相关性现状

根据实际数据，系统建议关注以下高相关对：

| 因子对 | 典型相关系数 | 建议 |
|--------|------------|------|
| 5日动量 ↔ 20日动量 | ~+0.85 | 保留20日，去掉5日 |
| PE ↔ EP | ~-0.92 | 保留EP（更稳定） |
| 换手率 ↔ 量比 | ~+0.78 | 保留换手率（更常用） |
| RSI14 ↔ 布林带位置 | ~+0.65 | 可保留（未超0.7） |

## 七、每日预计算架构

为了支撑看板秒开，系统采用 **"每日收盘后预计算 + 盘中读取快照"** 架构：

```
每日 16:30 (收盘后30分钟)
    │
    ▼
┌─────────────────────────┐
│  1. 加载全A股票池       │  ← 约4600只（已过滤ST/停牌/次新）
│  2. 批量拉取行情数据    │  ← 腾讯API + 多源降级
│  3. 计算15个因子        │  ← 全部截面Z-score标准化
│  4. 保存Parquet快照     │  ← data/daily_factors/
│  5. 写入meta.json       │  ← 元数据（因子数、耗时等）
└─────────────────────────┘
    │
    ▼
用户打开看板
    │
    ▼
┌─────────────────────────┐
│  直接读取当日快照        │  ← 0.1秒完成
│  不再实时计算            │  ← 避免20-30秒等待
└─────────────────────────┘
```

**快照文件：**

```
data/daily_factors/
  ├── factors_20260528.parquet  (35万条因子数据)
  ├── prices_20260528.parquet   (44万条价格数据)
  └── meta_20260528.json        (元数据)
```

**性能对比：**

| 模式 | 加载时间 | 适用场景 |
|------|---------|---------|
| 实时计算 | 20-30秒 | 历史回测、特殊分析 |
| 快照读取 | **0.1秒** | 日常看板、实时监控 |

## 十二、常见问题

**Q1: 为什么同样的因子，今天和昨天的得分不一样？**
> 得分是**截面相对值**（Z-score），取决于当日全市场的分布。即使个股因子值不变，市场整体变化也会导致得分变化。

**Q2: 信号强度8分一定比6分好吗？**
> 不一定。强度只代表该股票在**当日截面中的相对排名**。建议结合行情分类一起看：强势上涨中的高动量信号，比震荡市中的反转信号更可靠。

**Q3: 为什么有些股票没有信号？**
> 可能原因：1) 信号强度低于min_strength阈值；2) 被过滤出股票池（ST/停牌等）；3) 因子数据缺失（新股、长期停牌后复牌）。

**Q4: 如何调整策略风格？**
> 修改 `factor_weights` 参数可以调整风格：
> - 价值风格：加大PE/PB/EP权重
> - 成长风格：加大营收/利润增长率权重
> - 动量风格：加大momentum_20d权重
> - 反转风格：加大reversal/RSI权重

```
CacheManager工作流程：
1. 优先读取L2 Parquet缓存（按source/adjust隔离）
2. 可选读取L3 PostgreSQL缓存（需启用cache_l3_kline_enabled）
3. 缺失数据从FallbackFetcher拉取并写入缓存
```

### 6.3 回测引擎优化
- 回测引擎统一使用CacheManager获取数据，不再直接调用TencentFetcher
- 支持数据源自动降级：tencent → tushare → akshare → baostock
- 修复TencentFetcher股票代码转换，支持.SH/.SZ后缀格式

### 6.4 数据源配置

```env
# .env配置
```
