# 买卖点规则与回测撮合口径

> **版本**: v2.0
> **更新时间**: 2026-06-20 01:00 GMT+8
> **状态**: 当前强制执行口径
> **适用范围**: `quant-stock-picker` 信号页、策略回测页、回测记录页、K线复盘、参数网格验证
> **核心原则**: 低回撤优先、T日信号/T+1开盘执行、全链路可审计、禁止未来函数。

---

## 0. 给下一次 Agent 的必读结论

1. 当前系统已经不是 2026-06-11 的旧单规则买卖点口径。
2. K线复盘买卖点必须使用 **实际成交事件** `signals_executed`，禁止把未成交的 `signals_raw` 直接当成买卖点。
3. 所有买卖点必须区分：
   - `signal_date`: 原始规则在 T 日收盘后触发的信号日。
   - `exec_date`: T+1 开盘实际撮合日。
   - `date`: 对成交事件而言应等于实际成交日。
4. 当前主买点路径是：
   ```text
   大盘择时 → 因子截面评分 → L1趋势过滤 → L2策略匹配 → L3多条件共振 → T+1开盘执行
   ```
5. 当前主卖点路径是：
   ```text
   ATR硬止损 → 大盘防御减仓 → 策略失败退出 → ATR盈利保护型跟踪止盈/回撤止损 → ATR固定止盈 → 时间止损/最长持仓退出 → 信号卖出 → 末日清仓
   ```
6. P2 新增的短线退出规则可以在回测页关闭或调整参数：
   - `enable_market_defense_exit`
   - `enable_strategy_failure_exit`
   - `enable_time_stop`
   - `enable_max_holding_exit`
7. 历史软删除记录不要静默改写；旧记录的显示口径可能缺 `signal_date`，需要按记录中的实际字段解释。
8. 600143 历史排查结论：删除区中的新方案记录本身就是 `2026-06-03` 原始信号、`2026-06-04` 执行买入；若拿更早 `composite/均衡复合` 旧记录对比，则买点条件确实不同，因为旧记录是 legacy 布林单规则，新记录是三层过滤 + L3 共振。

## 1. 信号与成交的生命周期

```text
因子/行情数据
  ↓
策略评分与过滤
  ↓
signals_raw：T日收盘后的原始规则信号
  ↓
执行约束：T+1、资金、仓位、持仓、风控、成本、滑点
  ↓
signals_executed / trades：实际成交事件
  ↓
K线复盘、绩效统计、回测记录、退出审计
```

### 1.1 三类日期字段

| 字段 | 含义 | 用途 |
|---|---|---|
| `signal_date` | T 日收盘后产生信号的日期 | 审计信号来源，解释为什么买/卖 |
| `exec_date` | T+1 开盘实际撮合日期 | K线买卖点落点、交易流水成交日 |
| `date` | 事件日期 | 对 `signals_executed` / `trades` 应等于 `exec_date` |

### 1.2 K线复盘显示规则

当前强制规则：

```text
K线复盘买卖点 = signals_executed
K线点位日期 = exec_date
Tooltip / 明细保留 signal_date
```

禁止：

```text
直接把 signals_raw 当作实际买卖点展示
把 signal_date 当成交日
用 T 日收盘价模拟 T 日市价单
```

如果页面允许叠加原始规则信号，只能标为：

```text
原始信号 / 仅参考 / 不参与绩效统计
```

## 3. 买点规则

当前买点不是单指标触发，而是三层过滤。

源码：

```text
signals/layers.py
signals/scanner.py
backtest/scheme_backtest.py
```

### 3.0 买点专业化原则（2026-06-20 强制）

本项目不是为了拟合某一次回测胜率，也不是为了解释既有结果；唯一目标是用 A 股短线量化专业知识提高实盘买卖机会点准确度。

后续任何开仓/加仓规则调整必须遵循：

```text
1. 先定义交易假设，再选择指标。
2. 先整理开仓/加仓规则契约，再判断是否合理。
3. 不得看到单个 run 多/少一个买点就临时补规则拟合。
4. 不得把 RSI/BOLL/MACD/KDJ/量能等指标简单罗列，满足 N 个就交易。
5. 每个买点必须能说明：买点类型、主触发、确认项、否决项、风险标签、缺失数据。
6. TradePoint.confidence 是执行契约的一部分，不是展示字段。
```

`confidence` 当前事实：

```text
signals/layers.py 会生成 TradePoint.confidence。
20260620_172722_balanced 中：
  2026-03-25 BUY confidence=0.4903
  2026-03-31 BUY confidence=0.4205
  2026-06-03 BUY confidence=0.6603

当前 backtest/scheme_backtest.py 只把 confidence 传入并落盘，尚未用它决定是否开仓、仓位大小或是否允许加仓。
这是待修复的执行契约断裂。
```

专家规则方向：

```text
开仓：低置信度信号应观察或降仓，高置信度信号才允许标准开仓。
加仓：必须比开仓更严格；不能因为持仓中又出现 BUY 就加仓。
加仓至少应考虑：扣成本后盈利、原始买点结构未破、同路径再次确认、confidence 达到加仓阈值、单票仍 <=20%。
balanced：长期应作为 trend_momentum / pullback / breakout 的组合器或路由器，不应长期作为“任一形态 + 任意2/6指标共振即可交易”的独立策略。
```

### 3.1 Layer 1：趋势过滤

#### 非 `pullback`

要求偏趋势健康：

```text
Price > MA20
MA20 > MA40
ADX > 20
非20日最低
```

#### `pullback` / `balanced`

允许回调，但不能是纯弱势下跌：

```text
近10日曾经 Price > MA20
MA20 > MA40 加分
Price < MA20 允许，视为回调中
ADX > 20
非20日最低
```

通过阈值：

```text
score >= 0.4
```

### 3.2 Layer 2：策略形态匹配

| 策略 | 形态条件 |
|---|---|
| `trend_momentum` | 距20日高点 < 5%，5日动量 > 1%，20日动量 > 2% |
| `pullback` | 距20日高点回撤 > 5%，且 RSI < 40（偏弱回调，不等同标准超卖）或回撤 > 10% |
| `breakout` | 横盘振幅 < 8%，突破前期区间 1%，量比 > 1.3 |
| `balanced` | 任一子形态匹配即可 |

### 3.3 Layer 3：多条件共振

#### 默认 L3 条件全集

默认买入条件：

```text
1. rsi_weak_pullback  RSI < 40，偏弱回调；不应称为标准超卖
2. ma5_above_ma20     MA5 高于/金叉 MA20
3. macd_bullish       MACD 翻红
4. boll_lower         布林位置 < 0.3
5. volume_expand      放量 > 1.2x
6. kdj_golden         KDJ 金叉且 K < 50
```

RSI 专业口径：

```text
RSI < 20：极端超卖
RSI < 30：标准超卖
30 <= RSI < 40：偏弱回调 / 接近超卖，只能作为辅助确认
```

2026-06-20 修正：历史文案 `RSI超卖(39.6)` 已改为 `RSI偏弱回调：39.6 < 40`。买点 reason 会显式展示当前值、阈值和方向，并附加量能/成交额/换手率审计标签；换手率缺失时必须标记 `换手率缺失`，不得假装已验证。

默认卖出条件：

```text
1. rsi_overbought     RSI > 70
2. ma5_below_ma20     MA5 低于/死叉 MA20
3. macd_bearish       MACD 翻绿
4. boll_upper         布林位置 > 0.8
5. volume_price_down  放量下跌
```

## 4. 交易执行规则

源码：

```text
backtest/scheme_backtest.py
backtest/engine.py
```

### 4.0 历史回测 warmup 数据口径

```text
OHLCV 拉取必须按回测区间执行：
  start = backtest_start_date - warmup_days
  end   = backtest_end_date

warmup_days 默认不低于 120 天，避免 MA40 / ADX / MACD / RSI / BOLL / KDJ / ATR 在回测早期日期因前置K线不足而误判。
```

2026-06-20 证据：`600143` 在旧口径下 2026-03-20、2026-04-03 曾显示 L1 `数据不足`；修复后不再是数据不足，而是明确失败于 `近10日无上升趋势`。这是买点可信度审计的关键差异。

### 4.1 T+1 执行

```text
T 日收盘后产生信号
T+1 开盘执行
```

实现口径：

```python
next_date_map[signal_date] = next_trade_date
pending_actions[exec_date][symbol] = signal
exec_price = T+1 open ± slippage
```

严禁：

```text
用 T 日收盘价成交 T 日信号
用 T+1 收盘价作为 T+1 开盘成交
引用未来均线/未来行情判断 T 日信号
```

### 4.2 买入撮合

买入使用不复权开盘价：

```text
exec_price = open × (1 + slippage_rate)
```

A股整手：

```text
shares = floor(alloc / exec_price / 100) × 100
```

仓位硬约束：

```text
单票最大仓位 ≤ 20%
账户总仓位上限 ≤ 90%
```

### 4.3 卖出撮合

卖出使用不复权开盘价：

```text
exec_price = open × (1 - slippage_rate)
```

卖出成本：

```text
佣金 + 印花税 + 过户费 + 滑点
```

## 5. 成本与滑点

全局统一：

```text
佣金：双向 0.00025，最低 5 元
印花税：卖出单边 0.001
过户费：双向 0.00001
```

流动性滑点：

```text
成交额 > 5亿：0.002
1亿 ~ 5亿：0.005
成交额 < 1亿：0.010
```

所有回测必须内置四项成本：

```text
佣金
印花税
过户费
滑点
```

## 6. 卖点与退出规则

当前退出顺序固定如下，前面的规则先触发则当天不继续检查后面的规则。

```text
1. ATR硬止损
2. 大盘防御减仓
3. 策略失败退出
4. ATR盈利保护型跟踪止盈 / 跟踪回撤止损
5. ATR固定止盈
6. 时间止损 / 最长持仓退出
7. 信号卖出
8. 末日清仓
```

### 6.1 ATR硬止损

```text
stop_loss = avg_cost - stop_loss_atr_mult × ATR(14)
```

触发：

```text
T+1 开盘价 <= stop_loss
```

审计字段：

```text
exit_type = stop_loss
exit_subtype = atr_hard_stop
rule_name = ATR止损
```

### 6.2 大盘防御减仓

触发：

```text
market_score < market_defense_score
```

默认阈值：

```text
20
```

开关：

```text
enable_market_defense_exit
```

审计字段：

```text
exit_type = market_exit
exit_subtype = market_defense
rule_name = 大盘防御减仓
```

### 6.3 策略失败退出

开关：

```text
enable_strategy_failure_exit
```

观察窗口：

```text
failure_window_days
```

策略专属触发：

| 策略 | 失败条件 | exit_subtype |
|---|---|---|
| `trend_momentum` | 买入后短期跌破 MA20 | `trend_momentum_failed` |
| `pullback` | 跌破/接近20日低点，回调破位 | `pullback_breakdown` |
| `breakout` | 突破后跌回平台 | `breakout_failed` |

审计字段：

```text
exit_type = strategy_failure
rule_name = 动量失效退出 / 回调破位退出 / 突破失败退出
```

### 6.4 时间止损

用途：短线策略不允许长期占用资金；买入后 N 天仍未达到最低收益要求则退出。

开关：

```text
enable_time_stop
```

参数：

```text
time_stop_days
time_stop_min_profit_pct
```

触发：

```text
holding_days >= time_stop_days
且 projected_pct < time_stop_min_profit_pct
```

审计字段：

```text
exit_type = stop_loss
exit_subtype = time_stop
rule_name = 时间止损
```

示例：

```text
时间止损(7日收益-4.42%未达0.00%)
```

### 6.5 最长持仓退出

用途：满足短线定位，持仓不超过配置天数。

开关：

```text
enable_max_holding_exit
```

参数：

```text
max_holding_days
```

触发：

```text
holding_days >= max_holding_days
```

审计字段：

```text
exit_type = time_exit
exit_subtype = max_holding_days
rule_name = 最长持仓退出
```

### 6.6 ATR盈利保护型跟踪止盈

当前强制语义：

```text
亏损交易不能记录为 ATR跟踪止盈
```

必须先进入激活区间，才允许检查跟踪退出：

```text
highest >= avg_cost × (1 + trailing_activation_pct)
或
highest >= avg_cost + trailing_activation_atr_mult × ATR
```

默认参数：

```text
enable_trailing_exit = True
trailing_activation_pct = 0.05
trailing_activation_atr_mult = 1.0
```

激活后触发必须同时满足：

```text
open_price <= trailing_stop
pos.highest > avg_cost
trailing_stop > breakeven_sell_price
projected_pnl > 0
```

审计字段：

```text
exit_type = take_profit
exit_subtype = atr_trailing_profit
rule_name = ATR跟踪止盈
```

如果触发后实际扣成本为亏损，必须改写为：

```text
ATR跟踪回撤止损
exit_type = stop_loss
exit_subtype = atr_trailing_profit_failed
```

### 6.7 ATR固定止盈

触发：

```text
open_price >= take_profit
projected_pnl > 0
```

审计字段：

```text
exit_type = take_profit
exit_subtype = atr_fixed_profit
rule_name = ATR止盈
```

### 6.8 信号卖出

来自策略原始 SELL 信号，按 T+1 开盘执行。

审计字段：

```text
exit_type = signal_exit
exit_subtype = rule_signal
rule_name = 信号卖出 / 原规则名
```

### 6.9 末日清仓

回测最后一个交易日强制卖出所有剩余持仓。

审计字段：

```text
exit_type = final_liquidation
exit_subtype = end_of_backtest
rule_name = 末日清仓
```

## 9. 旧规则与当前规则的差异

### 9.1 旧 legacy 规则

旧版本常见信号：

```text
RSI反转
布林(20,2.0σ)
MA5/20交叉
MACD(12,26,9)
放量突破
KDJ交叉
```

旧记录中例如：

```text
composite / 均衡复合
布林下轨附近，位置9%
```

这是单规则触发，不等于当前三层过滤买点。

### 9.2 当前 layered 规则

当前新记录中例如 600143：
