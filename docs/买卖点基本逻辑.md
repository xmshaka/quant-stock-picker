# 买卖点基本逻辑规范

> **版本**: v1.0
> **生效日期**: 2026-06-11
> **状态**: 强制执行
> **关联**: DATA_INTEGRITY.md（数据红线）、BUG_0BUY_NSELL.md（历史教训）

---

## 一、核心原则

**任何策略在任何股票上，必须满足：有且仅有"先买后卖"的交易序列。**

- 没有买入，就不能卖出（禁止裸卖空）
- 每次卖出必须对应之前的某次买入
- 回测结果中 `买0卖N` 是 P0 级 Bug

---

## 二、信号生命周期

```
行情数据 → 规则评估 → 原始信号 → 过滤/去重 → 信号展示
                                    ↓
                              交易模拟引擎 → 实际执行交易
```

### 2.1 两套信号，用途不同

| 信号类型 | 来源 | 用途 | 存储位置 |
|---|---|---|---|
| **原始信号** | `evaluate_all_rules()` 规则评估 | K线图上显示买卖点标记，供用户分析 | `stock_signals` |
| **实际交易** | `_run_single_stock_backtest()` 模拟执行 | 计算收益、胜率、夏普等绩效指标 | `buy_count`/`sell_count`/`trades_pnl` |

两者可能不完全相同（实际交易受资金、持仓、T+1等约束），但原始信号是基础，必须独立保留。

---

## 三、信号生成规则（signals/rules.py）

### 3.1 规则评估入口

```python
evaluate_all_rules(bars, configs) → List[TradePoint]
```

**强制约束（在此函数中统一执行）：**

1. **先买后卖**：找到第一个 BUY，丢弃其之前的所有 SELL；若无 BUY，返回空列表
2. **同日同动作去重**：同一天同一个方向（BUY/SELL）只保留第一个
3. **按日期排序**：保证时间序列正确

### 3.2 单规则信号模式

所有规则统一采用**状态机/区域模式**，禁止精确穿越：

| 规则 | 买入触发 | 卖出触发 | 模式 |
|---|---|---|---|
| RSI反转 | RSI < 超卖线（30） | RSI > 超买线（70） | 状态机：进入区域触发一次，退出后重置 |
| 布林带 | 价格在下轨附近（位置≤10%） | 价格在上轨附近（位置≥90%） | 区域：进入区域触发一次 |
| MA交叉 | MA短上穿MA长 | MA短下穿MA长 | 穿越（本身就鲁棒） |
| MACD趋势 | 柱状线由负转正 | 柱状线由正转负 | 穿越（本身就鲁棒） |
| 放量突破 | 放量+上涨 | 放量+下跌 | 条件触发 |
| KDJ交叉 | K上穿D（超卖区） | K下穿D（超买区） | 穿越 |

**为什么不用精确穿越？** 实际行情中，RSI精确穿越30线的概率极低（60天可能0次），导致大量"0买入N卖出"的错误。状态机模式保证只要有超卖就有买入信号。

### 3.3 信号去重规则

```
evaluate_all_rules 内部：
  1. 各规则独立生成信号
  2. 合并后按日期排序
  3. 先买后卖过滤（丢弃首个BUY前的所有SELL）
  4. 同日同动作去重
```

## 四、交易执行规则（backtest/scheme_backtest.py）

### 4.1 T+1 规则

信号在 **T日收盘后** 产生，**T+1日开盘** 执行。

```python
# T+1 映射
next_date_map[signal_date] = date_list[date_list.index(signal_date) + 1]
```

**禁止用T日收盘价成交T日信号（未来函数）。**

### 4.2 持仓状态约束

| 条件 | 处理 |
|---|---|
| 收到 BUY 但已持仓 | 忽略（不加仓） |
| 收到 SELL 但未持仓 | 忽略（不裸卖空） |
| 同日既有 BUY 又有 SELL | SELL 优先执行（防当日冲销） |

### 4.3 末日清仓

回测最后一个交易日，强制卖出所有持仓（计算最终收益用）。标记为 `reason='末日清仓'`。

### 4.4 资金分配

```python
alloc = cash / max(1, len(symbols))  # 等分
shares = int(alloc / price / 100) * 100  # 向下取整到100股
```

### 4.5 手续费

| 项目 | 费率 |
|---|---|
| 佣金 | 万2.5（最低5元） |
| 印花税 | 千1（仅卖出） |
| 滑点 | 0.1% |

## 五、信号展示规则（dashboard/）

### 5.1 K线图展示

K线图上显示的是**原始信号**（`stock_signals`），不是实际执行的交易。

- 黄色三角 ▲ = 买入信号
- 紫色三角 ▼ = 卖出信号
- 来源：`evaluate_all_rules()` 的输出 + 末日清仓SELL

### 5.2 绩效指标

绩效指标（收益/夏普/回撤/胜率）来自**实际交易模拟**，与K线图上的信号标记可能不完全一致。

### 5.3 买卖计数校验

展示时自动校验：若某股票 `buy_count == 0 and sell_count > 0`，标记为异常（P0 Bug）。

---

## 六、多规则组合的信号冲突处理

当多个规则同时产生信号时，按以下优先级：

1. **同日同方向**：去重，保留第一个
2. **同日相反方向**：SELL 优先（保守策略，宁可错过也不被套）
3. **相邻日信号**：各自独立，不合并

---

## 七、边界情况

| 场景 | 处理 |
|---|---|
| 整个回测区间无 BUY 信号 | `stock_signals` 为空，不展示K线图 |
| 只有 BUY 没有 SELL | 末日清仓会补一个 SELL |
| 数据不足（<指标所需周期） | 跳过该规则，不生成信号 |
| 停牌日 | 无数据，自然跳过 |
| 新股上市（数据不足） | 该股票不参与信号评估 |

---

## 八、验证方法

回测完成后，自动执行以下校验：

```python
for sym, points in stock_signals.items():
    buy_count = len([p for p in points if p.action == 'BUY'])
    sell_count = len([p for p in points if p.action == 'SELL'])
    assert buy_count > 0 or sell_count == 0, f"BUG: {sym} 买{buy_count}卖{sell_count}"
```

## 十、资金管理与止盈止损（v1.1 新增）

### 10.1 资金管理

| 参数 | 默认值 | 说明 |
|---|---|---|
| `position_pct_per_entry` | 30% | 每次建仓/加仓占可用资金比例 |
| `max_single_pct` | 30% | 单只股票最大仓位占总权益比例 |
| `max_add_times` | 2 | 最大加仓次数（0=不加仓） |

**建仓**：`alloc = min(可用资金 × pos_pct, 总权益 × max_single_pct)`

**加仓**：`alloc = min(可用资金 × pos_pct, max_single_pct × 总权益 - 当前持仓市值)`

加仓后更新加权均价：`new_avg = (旧持仓成本 + 新买入成本) / 总股数`

### 10.2 ATR 止盈止损

| 类型 | 公式 | 触发价更新 |
|---|---|---|
| 止损 | `买入价 - 2×ATR(14)` | 建仓/加仓时重置 |
| 固定止盈 | `买入价 + 3×ATR(14)` | 建仓/加仓时重置 |
| 跟踪止盈 | `持仓最高价 - 2×ATR(14)` | 最高价创新高时更新（只升不降） |

**每日执行顺序**：
1. 更新 ATR + 持仓最高价 + 跟踪止盈
2. 开盘价检查：止损 → 跟踪止盈 → 固定止盈（按优先级）
3. 处理规则信号（BUY=建仓/加仓，SELL=信号卖出）
4. 计算当日权益

### 10.3 trade_details 字段

每笔交易记录：

```python
{
    'symbol': str,        # 股票代码
    'date': date,         # 交易日期
    'action': str,        # 'BUY' | 'SELL' | 'ADD'
    'price': float,       # 成交价
    'shares': int,        # 成交股数
    'cash_after': float,  # 成交后余额
    'reason': str,        # 原因
    # BUY/ADD 额外字段：
    'position_shares': int,  # 成交后总持股
    'avg_cost': float,       # 加权均价
    'stop_loss': float,      # 止损价
    'take_profit': float,    # 止盈价
    # SELL 额外字段：
    'cost': float,           # 买入均价
    'pnl': float,            # 盈亏金额
    'pnl_pct': float,        # 盈亏比例
    'holding_days': int,     # 持仓天数
    'entries': list,         # 建仓/加仓明细
}
```
